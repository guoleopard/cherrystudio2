# 模型名称输入方式变更说明

## 变更概述
已将模型名称的选择方式从下拉框改为手动输入，提供更灵活的模型配置体验。

## 主要变更内容

### 1. HTML 结构变更
- 将 `<select>` 下拉框改为 `<input>` 文本输入框
- 添加了 `datalist` 元素提供常用模型建议
- 保留了原有的 ID 和类名结构

### 2. CSS 样式更新
- 将 `.model-selector` 类名改为 `.model-input`
- 调整了输入框的宽度为 200px
- 添加了聚焦状态的样式效果

### 3. JavaScript 逻辑调整
- 将 `change` 事件改为 `input` 事件，实时响应输入
- 添加了 `blur` 事件处理，确保输入完成后保存设置
- 增强了模型名称验证和同步逻辑

### 4. 新增功能特性
- 支持手动输入任意模型名称
- 提供常用模型名称的自动补全建议
- 实时同步主界面和设置页面的模型输入
- 输入验证和错误处理

## 使用说明

### 主界面模型输入
1. 在工具栏的模型输入框中直接输入模型名称
2. 输入时会显示常用模型的建议列表
3. 支持任意有效的 OpenAI 模型名称

### 设置页面模型配置
1. 打开设置面板
2. 在"默认模型"输入框中输入模型名称
3. 输入完成后自动保存设置
4. 设置会自动同步到主界面

### 常用模型名称示例
- `gpt-3.5-turbo` - GPT-3.5 Turbo
- `gpt-4` - GPT-4
- `gpt-4-turbo` - GPT-4 Turbo
- `gpt-4-turbo-preview` - GPT-4 Turbo 预览版
- `gpt-3.5-turbo-16k` - GPT-3.5 Turbo 16K
- `gpt-4-32k` - GPT-4 32K
- `text-davinci-003` - Davinci 文本模型
- `text-curie-001` - Curie 文本模型

## 技术实现细节

### 事件处理
- `input` 事件：实时响应用户输入
- `blur` 事件：输入完成后保存设置
- 双向同步：主界面和设置页面实时同步

### 数据验证
- 自动去除输入内容的前后空格
- 验证模型名称是否为空
- 避免重复保存相同设置

### 用户体验优化
- 提供模型名称建议列表
- 输入框聚焦状态视觉反馈
- 平滑的过渡动画效果

## 兼容性说明
- 完全兼容原有的设置存储格式
- 支持所有 OpenAI API 模型
- 向后兼容，不影响现有功能

## 注意事项
- 请确保输入的模型名称正确无误
- 某些模型可能需要特定的 API 权限
- 模型名称区分大小写，建议按照官方格式输入
